# アーキテクチャ

NoraBar は WPF の MVVM 構成を基礎とし、Phase 0 で HUD の選択と表示をモジュール化しています。`App` が Composition Root、`HudRouter` が選択ライフサイクル、`MainWindow` が汎用シェルを担当します。既存の `MainViewModel` は音楽 View の互換 DataContext として維持されています。

## 全体構成

```text
App（Composition Root）
  ├─ MainViewModel → MusicViewModel → MediaControl / AudioVisualizer / Lyrics
  ├─ MusicHudModule : IHudModule
  ├─ HudRegistry
  ├─ HudRouter
  └─ MainWindow
       ├─ HudRouter.StateChanged / PresentationChanged
       ├─ 現在モジュールの View をホスト
       └─ 位置、DPI、入力、アニメーション、設定、トレイ、終了
```

`App.xaml` は `StartupUri` を持たず、`ShutdownMode="OnExplicitShutdown"` を指定します。`App.OnStartup` が依存を明示的に生成し、constructor injection で接続します。DI コンテナは使用していません。

## HUD ID と PresentationState

HUD は安定した文字列 ID で識別します。組み込み ID は `BuiltInHudIds` に集約され、音楽 HUD は `BuiltInHudIds.Music`（値は `music`）です。`HudRegistry` と Router の ID 判定は大文字・小文字を区別する Ordinal 比較です。

`HudPresentationState` は HUD 選択と独立し、`Collapsed`、`Peek`、`Expanded`、`Pinned` を持ちます。`SetPresentationState` と `CollapseFromPointerLeave` は表示状態だけを変更し、`ActivateAsync` と `DeactivateAsync` を呼びません。同じ状態、初期化前、遷移中、Shutdown 開始後の要求は拒否され、通知も発行されません。`Pinned` はマウス離脱で折りたたみません。

## IHudModule

`IHudModule` は HUD の識別情報、選択ライフサイクル、View、推奨サイズを共通化する WPF 向け契約です。

```csharp
public interface IHudModule : IAsyncDisposable
{
    string Id { get; }
    HudModuleMetadata Metadata { get; }
    event EventHandler? PresentationInvalidated;
    ValueTask InitializeAsync(CancellationToken cancellationToken);
    ValueTask ActivateAsync(CancellationToken cancellationToken);
    ValueTask DeactivateAsync(CancellationToken cancellationToken);
    FrameworkElement GetView(HudViewContext context);
    HudSize GetPreferredSize(HudViewContext context);
}
```

`InitializeAsync` は初回準備、`ActivateAsync` は現在 HUD への選択、`DeactivateAsync` は切り替えまたは終了を表します。View やサイズへ影響する状態変化は `PresentationInvalidated` で通知します。`HudViewContext` は表示状態を渡し、`HudSize` は推奨幅と高さを返します。ライフサイクルと `DisposeAsync` は可能な限り冪等にします。

## HudRegistry

`HudRegistry` は `IHudModule` を手動登録し、Ordinal ID 検索、重複・不正 ID の拒否、登録順列挙、全登録モジュールの一度だけの非同期破棄を提供します。登録と破棄開始を同期し、複数の破棄失敗は集約して報告します。アセンブリスキャンや外部 DLL 読み込みは行いません。

## HudRouter

`HudRouter` は実行時の有効 HUD、実効既定 HUD、現在 HUD、表示状態、選択ライフサイクルを一元管理します。`PresentationInvalidated` の購読所有者は Router だけで、汎用の `PresentationChanged` に変換します。

### 一貫したスナップショット

`GetSnapshot()` は短い状態ロックで関連値を一つの不変値として取得します。

```csharp
public readonly record struct HudRouterSnapshot(
    string? CurrentHudId,
    IHudModule? CurrentModule,
    HudPresentationState PresentationState,
    bool IsInitialized,
    bool IsShuttingDown);
```

初期化前と終了後は ID とモジュールが `null` です。`MainWindow` は通知ごとにスナップショットを一度だけ取得し、そこから `HudViewContext` を作り、同じモジュールに `GetView` と `GetPreferredSize` を問い合わせます。

### 同期と通知

`InitializeAsync`、`NavigateToAsync`、`DisableAsync`、`ShutdownAsync` は `SemaphoreSlim` で直列化します。状態読み書きは短い `lock` で同期し、`StateChanged` と `PresentationChanged` はロック外で発行します。同期 API の表示状態変更も同じロックで遷移・終了との競合を防ぎます。

`ActivateAsync` 中の無効化通知は保留し、現在 HUD 確定と `StateChanged` の後に複数回分を一つの `PresentationChanged` へ統合します。未登録 ID は実行時有効一覧から除外し、有効 HUD がなくなれば `music`、それも未登録なら登録順の先頭を補完します。この実行時補正は保存設定を書き換えません。

## MusicHudModule

`MusicHudModule` は最初の組み込みモジュールで、Minimal、Productivity、Lyrics Focus の View 選択とデザイン別キャッシュ、互換 DataContext、`ShowProgressBar`・`ShowLyrics`・`HasMultipleSessions` によるサイズ計算、表示変更監視、WPF Dispatcher 整合性、Cleanup を所有します。

設定変更ごとに View を再生成せず、キャッシュ済み View と推奨サイズを再評価します。`InitializeAsync` は購読を一度だけ構成し、`DisposeAsync` は購読解除と進行中通知の完了を待ってから Cleanup を一度だけ実行します。`MusicViewModel.Cleanup` の直接所有者は `MusicHudModule.DisposeAsync` だけです。

## MainWindow

`MainWindow` は、HUD の View 選択と推奨サイズについては `StateChanged` と `PresentationChanged` だけを監視する汎用ホストです。モジュールを直接購読せず、一つの Router スナップショットで View とサイズを再評価します。表示言語や位置編集など、シェル固有状態は引き続き `MainViewModel` から監視します。

画面上端配置、マルチモニター、DPI、入力、共通 `200x2` 折りたたみ、アニメーション、全画面抑制、位置編集、言語、設定、トレイ、終了を担当します。音楽 View 型、`DesignVariant`、`ShowLyrics`、`ShowProgressBar`、`HasMultipleSessions`、HUD ID 分岐は持ちません。

Alt+F4、システムメニュー、OS Close も `OnClosing` で App の最終 Close 以外をキャンセルし、重複防止された終了アニメーションと共通終了へ転送します。

## 起動ライフサイクル

```text
App.OnStartup
  → MainViewModel → MusicHudModule → HudRegistry登録 → HudRouter
  → MainWindow生成・StateChanged / PresentationChanged購読
  → HudRouter.InitializeAsync
       → 実効既定HUD解決
       → InitializeAsync（初回のみ）
       → PresentationInvalidated購読
       → ActivateAsync
       → CurrentHudId / CurrentModule / Collapsed確定
       → StateChanged → 必要ならPresentationChanged
  → MainWindow初回評価 → Show
```

Router 初期化前に MainWindow が購読するため初期通知を取りこぼしません。Mutex、`--startup`、通常起動時の設定画面、更新確認は維持します。

## HUD 切り替えライフサイクル

```text
NavigateToAsync(newHudId)
  → 遷移先検証・フォールバック
  → 旧購読解除 → 旧DeactivateAsync
  → 新InitializeAsync（初回のみ）→ 新購読
  → 新ActivateAsync（通知保留）
  → CurrentHud確定 → StateChanged
  → 保留があればPresentationChangedを一度発行
  → MainWindow再評価
```

新購読は Activate 前、旧購読解除は Deactivate 前です。失敗時は新モジュールを後始末し、旧モジュール、実効既定モジュールの順で復旧します。復旧不能なら現在 HUD を `null`、表示を `Collapsed`、初期化状態を false にして `HudNavigationException` を送出します。現在 HUD の `DisableAsync` は更新後一覧から先にフォールバックを解決して切り替えます。

## 終了ライフサイクル

```text
ウィンドウ / メニュー / トレイの終了要求
  → MainWindow終了アニメーション
  → App.RequestShutdownAsync（同時要求を同じTaskへ統合）
  → MainWindow.DetachHudRouter
  → HudRouter.ShutdownAsync
       → 現在モジュール購読解除 → DeactivateAsync → CurrentHudをnull
  → HudRegistry.DisposeAsync
       → 全モジュールを一度ずつDisposeAsync
       → MusicHudModuleがCleanupを一度実行
  → シェル資源解放 → AllowClose → Close → Application.Shutdown
```

`App.OnExit` は Mutex だけを破棄し、新しい非同期終了を開始しません。`MainWindow.OnClosed` は冪等な最終解放だけを行います。MainWindow、Router、App.OnExit は Cleanup を直接呼びません。

## 設定 SchemaVersion と移行

`UserSettings` は既存設定に `SchemaVersion`（現在値 1）、`DefaultHudId`（`music`）、`EnabledHudModuleIds`（`["music"]`）、`Modules`、未知トップレベル JSON 用 `AdditionalProperties` を追加しています。

`UserSettingsJson` は純粋な構造正規化を行います。SchemaVersion なし・0・負数だけを現在値へ補完し、現在より大きい未来版は保持します。未登録 ID、未知モジュール JSON、未知トップレベル JSON も往復保持します。正規化と Serialize は新しい設定、コレクション、複製した `JsonElement` を使い、呼び出し元を変更しません。壊れた JSON は既定値へフォールバックし、`SettingsService` は `%AppData%\NoraBar\settings.json` と旧配置からの移行を維持します。

設定層は未知 ID や空一覧を保持し、実行可能性の補正は Router だけが行うため、未来版や一時的に利用できない HUD のデータを失いません。

## 組み込み HUD の追加手順

安定 ID を `BuiltInHudIds` に追加し、`IHudModule` を実装して `App.OnStartup` で登録します。設定の有効一覧と必要なら既定 ID を更新し、ライフサイクル、フォールバック、View、サイズ、破棄をテストします。

```csharp
public sealed class ClockHudModule : IHudModule
{
    private readonly FrameworkElement _cachedView = new ClockHudView();

    public string Id => BuiltInHudIds.Clock;
    public HudModuleMetadata Metadata { get; } = new("Clock", 10);
    public event EventHandler? PresentationInvalidated;
    public ValueTask InitializeAsync(CancellationToken token) => ValueTask.CompletedTask;
    public ValueTask ActivateAsync(CancellationToken token) => ValueTask.CompletedTask;
    public ValueTask DeactivateAsync(CancellationToken token) => ValueTask.CompletedTask;
    public FrameworkElement GetView(HudViewContext context) => _cachedView;
    public HudSize GetPreferredSize(HudViewContext context) => new(320, 96);
    public ValueTask DisposeAsync() => ValueTask.CompletedTask;
}

registry.Register(new ClockHudModule());
```

```json
{
  "DefaultHudId": "clock",
  "EnabledHudModuleIds": ["music", "clock"],
  "Modules": { "clock": {} }
}
```

`BuiltInHudIds.Clock`、`ClockHudModule`、Clock View は説明用で、Phase 0 には未実装です。

## Phase 0 の外部プラグイン境界

```text
外部プラグイン読み込み（将来）
  → IHudModule → HudRegistry → HudRouter → MainWindow
```

実装済み境界は `IHudModule`、Registry、Router、汎用ホストまでです。外部 DLL ローダー、アセンブリスキャン、フォルダー監視、分離ロード、署名・信頼管理、プラグインストアは未実装です。独立 SDK と外部バイナリ互換性も保証しません。上部タブ、新しい実用 HUD、ライブアクティビティ、自動切り替えも Phase 0 外です。

## 既存 ViewModel と Services

`MainViewModel` は設定・シェル状態と音楽 View の互換 DataContext、`MusicViewModel` は曲・再生・歌詞・波形・操作を提供します。Windows メディア連携は `MediaControlService`、音声解析は `AudioVisualizerService`、歌詞は `LyricsService`、保存は `SettingsService`、自動起動は `StartupService`、言語は `LocalizationService` が担当します。
